Welcome to MusicLens

A simple web application that uses Musicbrainz's SPARQL endpoint to run search queries on the music RDF store.

The endpoints can be found at:

  • http://dbtune.org/musicbrainz/sparql
  • http://linkedbrainz.org/sparql

Musicbrainz stores meta data about music. This includes meta data about artists, albums, and tracks. We can query the database to find information like tracks in a particular album, top sold albums by a particular band/artist, whether a person belongs to a certain band or not, etc.

Why MusicLens?

A regular user who doesn't have knowledge about RDF and SPARQL will not be able to query the Musicbrainz's RDF store. Sure, he can get any information related to a certain artist or track on Google. But, he'll have to browse through the search results, and sometimes dig in deeper. With MusicLens, the user can query Musicbrainz data without knowing how the queries are written. Right now, we're providing static queries in our application. With time, we plan to implement dynamic queries which will take input from the user and output the results accordingly.

Some generalized queries:

  • List URI of a particular artist/band
  • List members of a band
  • List top trakcs of a band/artist
  • Is this person member of some band?
  • Does this band exist?
  • How many bands of the same name exist?
